Fix the coach login and role flow in the Pllay On app.
Currently when a user selects "COACH" role they are taken to the
athlete registration form. This is wrong.

Do not change any existing athlete functionality.

THE PROBLEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Current flow (broken):
  Select COACH role → athlete registration form → athlete dashboard

Required flow:
  Select COACH role → coach registration form → coach dashboard
  Select ATHLETE role → athlete registration form → athlete dashboard
  Select PARENT role → parent registration form → parent dashboard

The role selected at login must gate the entire experience from
registration through to dashboard. Not just a UI toggle — a
fundamentally different experience per role.

ROLE ROUTING LOGIC
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

On login/register, after role is selected and stored in localStorage
as 'po_role', route as follows:

  po_role === 'athlete' → /register/athlete → /dashboard/athlete
  po_role === 'coach'   → /register/coach   → /dashboard/coach
  po_role === 'parent'  → /register/parent  → /dashboard/parent

Each role has its own registration form, its own dashboard,
and its own navigation structure.

The athlete_id that links coach and parent to an athlete is set
during their registration (they enter the athlete's name/email
to link to the correct athlete account).

COACH REGISTRATION FORM — /register/coach
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Cover style: dark background (#111111), blue (#0B7DF1) accent
"PLLAY ON" brand top left in green
"COACH REGISTRATION" in Barlow Condensed, blue

Fields:
  Coach Name         (text, required)
  Academy / Club     (text, required)
  Specialisation     (text, e.g. "Tennis — Junior Development")
  Email              (email, required)
  Password           (password, required)

Section divider: "LINK TO ATHLETE"
  Athlete Email      (text — email of the athlete account to link to)
  hint: "Enter your athlete's email address. They must already
         have a Pllay On account."

"REGISTER AS COACH" button in blue (#0B7DF1)

On submit:
  1. Create Supabase auth account
  2. Insert into coaches table (see DB below)
  3. Look up athletes table by athlete_email to get athlete_id
  4. If found: link coach to athlete via coach_athlete_links table
  5. If not found: show message "Athlete account not found.
     Ask your athlete to register first, then come back."
  6. On success: redirect to /dashboard/coach

DATABASE:

create table coaches (
  id uuid primary key references auth.users(id),
  created_at timestamptz default now(),
  name text not null,
  club text,
  specialisation text,
  email text
);

create table coach_athlete_links (
  id uuid primary key default gen_random_uuid(),
  coach_id uuid references coaches(id) on delete cascade,
  athlete_id uuid references athletes(id) on delete cascade,
  linked_at timestamptz default now(),
  unique(coach_id, athlete_id)
);

alter table coaches enable row level security;
alter table coach_athlete_links enable row level security;

create policy "coach_own" on coaches
  using (auth.uid() = id);
create policy "coach_links_own" on coach_athlete_links
  using (auth.uid() = coach_id);
