ARCHITECTURE — TWO REFLECTION TYPES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WEEKLY REFLECTION (existing — mandatory)
  - One per week
  - Unlocks each Sunday
  - Must be completed before Week N+1 unlocks
  - Contains: scorecard, 3 reflection fields, focus question,
    implementation intention
  - Completion marks the week as done
  - Already built — do not change the content or logic

DAILY REFLECTION (new — optional)
  - Available every day of the week
  - Athlete can add one per day (Monday through Sunday)
  - Takes 2 minutes maximum
  - Focused on that specific training session
  - Does NOT affect week completion status
  - Does NOT unlock or lock anything
  - Purely additive — more data for the athlete to review

DATABASE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

create table daily_reflections (
  id uuid primary key default gen_random_uuid(),
  athlete_id uuid references athletes(id) on delete cascade,
  week_number int not null check (week_number between 1 and 12),
  day_of_week text not null,
  session_type text,
  session_focus text not null,
  went_well text,
  challenging text,
  development_note text,
  physical_status text,
  session_rating int check (session_rating between 1 and 5),
  energy_rating int check (energy_rating between 1 and 5),
  created_at timestamptz default now(),
  updated_at timestamptz default now(),
  unique(athlete_id, week_number, day_of_week)
);

alter table daily_reflections enable row level security;

create policy "daily_own" on daily_reflections
  using (auth.uid() = athlete_id);

DESIGN RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  - "MANDATORY" and "OPTIONAL" badges make the distinction
    immediately clear without any explanation needed
  - Weekly review: phase colour dominant throughout
  - Daily logs: near-white (#F8FAFC) background — deliberately
    lower visual weight than the weekly review
  - The weekly review must feel more important than the daily logs
    at a glance — hierarchy through colour and weight, not text
  - No motivational language anywhere
  - Space Mono for all labels and badges
  - Barlow Condensed for all section headers
  - Inter for all body text and placeholder text
  - Mobile-first — modal bottom sheet works on phone screen
  - All touch targets minimum 44px
