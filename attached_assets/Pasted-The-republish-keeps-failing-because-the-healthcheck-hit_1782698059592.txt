The republish keeps failing because the healthcheck
hits /api before the server is ready to respond.

This needs a fundamental fix to how the server
starts and how the healthcheck is configured.

THE REAL PROBLEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The logs show this sequence on every deploy:

  1. starting artifact process port=8080
  2. healthcheck failed /api → status 500
  3. artifact process started pid=19
  4. healthcheck failed /api → status 500 (×5)
  5. upsertUser crashes trying to insert users

The healthcheck fires at the SAME millisecond
as the process starts — before Express has
even registered routes. The server cannot
possibly respond in time.

FIX 1 — CONFIGURE THE HEALTHCHECK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Find the Replit deployment configuration file.
It will be one of:
  .replit
  replit.nix
  deployment.json

Open .replit and find or add a [deployment]
section. Set the healthcheck path and add
a startup delay:

  [deployment]
  deploymentTarget = "autoscale"
  ignorePorts = false
  build = ["npm", "run", "build"]
  run = ["node", "artifacts/api-server/index.js"]

  [[ports]]
  localPort = 8080
  externalPort = 80

Check if there is a healthCheckPath setting
and change it to use a simpler endpoint:

  healthCheckPath = "/health"

Then in the Express server, add this route
as THE VERY FIRST ROUTE before everything else:

  // Must be first — before any middleware
  app.get('/health', (req, res) => {
    res.status(200).send('ok')
  })

  app.get('/api', (req, res) => {
    res.status(200).json({ status: 'ok' })
  })

These two lines must appear before:
  - Session middleware
  - Database connections
  - Any other app.use() calls
  - ensureSchema()
  - Any route imports

FIX 2 — MAKE SERVER START SYNCHRONOUSLY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The server must call app.listen() BEFORE
doing anything async. Rewrite the startup
sequence:

  import express from 'express'

  const app = express()
  const PORT = process.env.PORT || 8080

  // STEP 1 — Register health routes immediately
  // Nothing before these two lines
  app.get('/health', (req, res) => {
    res.status(200).send('ok')
  })

  app.get('/api', (req, res) => {
    res.status(200).json({
      status: 'ok',
      app: 'Pllay On Edge'
    })
  })

  // STEP 2 — Start listening immediately
  // Do not await anything before this
  app.listen(PORT, '0.0.0.0', () => {
    console.log(`Server listening on port ${PORT}`)
    // STEP 3 — Run everything else AFTER listening
    initializeApp()
  })

  // All async setup goes in here
  async function initializeApp() {
    try {
      // Register remaining middleware
      app.use(express.json())
      app.use(express.urlencoded({ extended: true }))

      // Session setup
      // ... session middleware here ...

      // Database schema
      await ensureSchema()
      console.log('Schema ready')

      // All API routes
      // ... route registrations here ...

      // Static files last
      app.use(express.static('dist'))
      app.get('*', (req, res) => {
        res.sendFile('dist/index.html',
          { root: process.cwd() })
      })

      console.log('App fully initialised')
    } catch (err) {
      console.error('Init error:', err.message)
      // Do not crash — server stays up for healthcheck
    }
  }

This guarantees:
  - /health responds in <10ms
  - /api responds in <10ms
  - Healthcheck passes immediately
  - Database setup runs after
  - Deploy succeeds

FIX 3 — UPSERT USER CRASH
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The logs still show:
  Error: Failed query: insert into "users"
  at async upsertUser in auth.ts

The users table creation must happen in
ensureSchema() BEFORE any auth routes
process requests.

In ensureSchema(), add this as the FIRST
table to create:

  await db.execute(sql`
    CREATE TABLE IF NOT EXISTS users (
      id TEXT PRIMARY KEY,
      email TEXT NOT NULL UNIQUE,
      first_name TEXT,
      last_name TEXT,
      profile_image_url TEXT,
      created_at TIMESTAMP DEFAULT NOW(),
      updated_at TIMESTAMP DEFAULT NOW()
    )
  `)

Also in auth.ts, wrap upsertUser completely:

  export async function upsertUser(userData: any) {
    try {
      if (!userData?.id || !userData?.email) {
        console.error('upsertUser: missing id or email')
        return null
      }
      const result = await db
        .insert(users)
        .values({
          id: userData.id,
          email: userData.email,
          firstName: userData.firstName || null,
          lastName: userData.lastName || null,
          profileImageUrl: userData.profileImageUrl || null,
          updatedAt: new Date()
        })
        .onConflictDoUpdate({
          target: users.id,
          set: {
            email: userData.email,
            firstName: userData.firstName || null,
            lastName: userData.lastName || null,
            updatedAt: new Date()
          }
        })
        .returning()
      return result[0] || null
    } catch (error: any) {
      console.error('upsertUser failed:', error.message)
      return null
    }
  }
