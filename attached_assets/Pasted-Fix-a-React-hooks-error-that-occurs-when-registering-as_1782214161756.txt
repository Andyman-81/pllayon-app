Fix a React hooks error that occurs when registering as a coach.

The error is:
"Rendered more hooks than during the previous render"

This appears on the /register/coach page when the registration
form is submitted or when conditional fields appear/disappear.

WHAT CAUSES THIS ERROR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

React requires that hooks (useState, useEffect, useCallback etc)
are always called in the same order on every render.

The error means somewhere in the coach registration component
(and possibly parent registration too) a hook is being called
inside a conditional block, like:

  // WRONG — hook inside a condition
  if (someCondition) {
    const [value, setValue] = useState('')  // ← breaks React rules
  }

Or a hook is called after an early return:

  // WRONG — hook after early return
  if (!data) return <Loading />
  const [value, setValue] = useState('')  // ← never reached on first render

THE FIX — RULES TO APPLY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Open every file related to registration and auth.
Apply these fixes:

RULE 1 — Move ALL hooks to the top of the component.
All useState, useEffect, useCallback, useMemo calls must be
the very first thing in the component function body.
No exceptions. Before any conditionals, before any early returns.

RULE 2 — Remove all early returns that appear before hooks.
If you need to show a loading state or redirect, do it with
conditional rendering in the JSX return — not with an early return
before the hooks.

WRONG:
  function CoachRegister() {
    if (loading) return <Spinner />  // early return BEFORE hooks
    const [name, setName] = useState('')  // hook AFTER early return
  }

CORRECT:
  function CoachRegister() {
    const [name, setName] = useState('')  // hook FIRST
    const [loading, setLoading] = useState(false)
    if (loading) return <Spinner />  // early return AFTER all hooks
  }

RULE 3 — Do not call hooks inside conditionals.

WRONG:
  if (role === 'coach') {
    const [club, setClub] = useState('')
  }

CORRECT:
  const [club, setClub] = useState('')
  // then conditionally RENDER the field, not conditionally DECLARE the hook

RULE 4 — Do not call hooks inside loops or nested functions.
All hooks at the top level of the component only.

      // any setup

    // Handler functions

If there is a shared RoleSelector component that conditionally
renders different registration forms based on the selected role,
check that it does not remount components when switching roles.

If switching from Athlete to Coach causes a full component remount
(destroying and recreating the component), this can also cause
the hooks count mismatch error.

Fix: use a stable parent component that always renders all three
forms but shows/hides them with CSS display:none — do not
conditionally mount/unmount the form components based on role.

OR use React.memo and stable keys to prevent unnecessary remounts.
