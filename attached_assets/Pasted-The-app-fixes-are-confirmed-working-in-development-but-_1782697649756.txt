━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The most common cause of Replit republish
failure after code changes is the health check
timing out during deployment.

The health check hits /api immediately when
the server starts. If the server takes more
than a few seconds to be ready (because it
is running database migrations), the health
check fails and deployment is marked as failed.

Fix: delay the schema migration so the server
responds to /api before running migrations.

In the server entry file, restructure startup:

  // 1. Start listening FIRST
  const server = app.listen(PORT, '0.0.0.0', () => {
    console.log(`Server listening on port ${PORT}`)
  })

  // 2. Run migrations AFTER server is ready
  server.on('listening', async () => {
    try {
      await ensureSchema()
      console.log('Schema ready')
    } catch (err) {
      console.error('Schema migration failed:', err.message)
      // Do not crash — server keeps running
    }
  })

This way the health check can hit /api immediately
while migrations run in the background.
